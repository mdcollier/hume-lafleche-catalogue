Experimental studies & methods

A public-safe index of the experimental workflow behind the La Flèche / Riboulet project.
Claim ceiling. This site presents approved source previews, catalogue materials, computational routing metadata, and review records to support scholarly inspection. It does not claim that Hume read, used, borrowed from, contacted, depended on, or was influenced by any listed text.

This page organizes the kinds of experimental work the project has carried out — retrieval and ranking, embeddings and similarity, catalogue analytics, OCR and source preparation, review-zone and interface studies, and validation / null / sensitivity checks. It is an index of the methods, not a results dashboard.

Everything here is metadata-only and describes experimental project workflow and scholarly-review support. Detailed results, scores, ranked lists, and pair-level material are internal working outputs and are not published on this public site by design. Nothing here states or implies Humean influence, contact, source-dependence, reading history, discovery, validation, recall, or coverage. Retrieval / routing / embedding / similarity / comparison / analytics / validation / review are used only as method terms.

Documentation. Each entry links to a public-safe method note summarized from the project's internal documentation, and to the public page that best documents it where one exists. The working repository is not public, so detailed records — ranked queues, per-text metrics, scores, and model outputs — are internal working outputs and are not published on this public site by design.

Status labels are descriptive notes, not formal categories: available — a public surface on this site · frozen baseline — a fixed reference configuration · complete — the study was run and concluded · evaluated — assessed, with the decision recorded · design — design / feasibility stage · doctrine — a settled definition or policy · protocol — a defined working procedure · feature layer — a supporting signal, never a gate.

Retrieval and routing

Frozen retrieval pipeline frozen baseline
Purpose: surface candidate comparison material into ordered review queues. Method: sentence-bounded spans, a dense multilingual-embedding channel plus local lexical channels (IDF and character n-grams), fused by reciprocal-rank fusion; parent-paragraph scoring under a fixed rubric. Status: the project's frozen baseline, applied uniformly to every text.
Documentation: Method note · Catalogue coverage & folders.
Retrieval-channel ablation / evaluation replay complete
Purpose: test how much each retrieval channel and reranking variant contributes. Method: replay over a fixed evaluation set with per-channel ablations and rerank variants. Status: complete. Public-safe takeaway: channel contributions are language-dependent (lexical channels help some cases and add noise in others).
Documentation: Method note.
Section-aware / wide-net surfacing feasibility design
Purpose: assess surfacing material beyond a fixed top-K cut. Method: section-aware wide-net screening design as an additive extension, not a replacement of the frozen baseline.
Documentation: Method note.

Embeddings and similarity

Alternative-embedding / competitor-model arc evaluated
Purpose: evaluate alternative dense embedding models as candidate retrieval channels. Method: compared open and proprietary embedding models against the full frozen production stack on known terrain. Public-safe takeaway: no tested alternative model outperformed the full production stack on known terrain; one model was selectively ratified as a triggered auxiliary lens under specific conditions, never as evidence by itself. This is model / channel evaluation, not a finding about Hume's sources.
Documentation: Method note.
Register-gap / cross-language similarity checks complete
Purpose: measure the representational limits of paragraph-level similarity across languages and registers. Method: doctrine-matched cross-language comparison tests. Public-safe takeaway: some material is not visible at the paragraph-similarity layer — a representational limit of the method, not an OCR or contamination problem.
Documentation: Method note.

Catalogue analytics

Catalogue coverage & representation analytics available
Purpose: track which Riboulet catalogue sections are represented in project records and which are not yet searched. Method: section-level aggregation over committed project records. Status: a public-safe surface.
Documentation: Method note · Catalogue sections & folders · Coverage map.
Convergence diagnostics feature layer
Purpose: additive, read-only diagnostics over pre-cut convergence patterns. Method: distinctiveness-ranked sidecars that annotate but never delete or reorder review queues.
Documentation: Method note.

OCR and source preparation

OCR & normalized-project-text preparation available
Purpose: produce readable working source layers from page images. Method: OCR followed by deterministic normalization — paragraph reflow, de-hyphenation, page-anchor preservation, and removal of markup and OCR-engine artifacts; no paraphrase, translation, or invented text. Status: the four public previews are the readable output layer.
Documentation: Method note · Source previews.
Source-faithful vs normalized text-layer taxonomy doctrine
Purpose: define which text layers are appropriate for public display. Method: a layered file / quality model distinguishing source-faithful, page-verified, and normalized reading layers, each with visible caveats. Status: the public previews carry the normalized-working-text label and its caveats.
Documentation: Method note · Source previews.

Review zones and interface studies

Review-zone interface available
Purpose: index metadata-only comparison / review records for scholarly inspection. Method: a section- and zone-oriented metadata interface; no source text. Status: available.
Documentation: Method note · Review zones.
Cheap-tier attention / promoter studies feature layer
Purpose: test whether inexpensive model-generated metadata can help order review queues. Method: cheap-model screening calibrated against panel and registry references. Public-safe takeaway: used only as an ordering / attention signal — never as a gate, deletion rule, or deprioritization of protected material.
Documentation: Method note.
Direct-reading adjudication harness protocol
Purpose: a disciplined close-reading adjudication step before anything is promoted for review. Method: a leak-scanned direct-read protocol with a hard pre-commit safety gate.
Documentation: Method note.

Validation, null, and sensitivity checks

Negative-control / null studies complete
Purpose: guard against spurious signal. Method: negative-control text selection and blind checks run alongside the main workflow. Public-safe takeaway: null and negative controls are a standing part of the methodology.
Documentation: Method note.
English-comparator calibration arc complete
Purpose: calibrate the instrument against documented English comparison texts. Method: development / validation / held-out splits under the frozen architecture. Public-safe takeaway: calibration evidence for evaluating the method — not a general rule and not a finding about Hume's sources.
Documentation: Method note.
Benchmark / cross-cycle synthesis complete
Purpose: compare method cycles against a benchmark reference registry. Method: registry-anchored evaluation across cycles.
Documentation: Method note.

Public-site / presentation experiments

Public source-preview & catalogue interface development available
Purpose: build claim-safe, scholar-facing public surfaces. Method: static exporters plus iterative information-architecture work for the source previews and catalogue views. Status: this site is the current output.
