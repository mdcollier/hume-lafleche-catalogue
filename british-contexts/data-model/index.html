Public data model

The British Contexts public content is built from a structured, versioned data layer: texts, relation records, evidence/passage records, retrieval-metric records, run provenance, reports, and releases. Passages and scores are display-gated records, shown only when explicitly authorised.

Counts and display states only on this page; no passage text or raw scores are shown here.

Record layers

Runs 1 0 Technical provenance for metrics — embedding model, index, chunking, fusion parameters, code commit.
Reports 4 4 Public-safe report/audit summaries — satellite review, audits, validation synthesis, relation-status metadata.
Releases 4 4 Release history — what was added/updated/withdrawn, validation result, validator version.

Display states

  • Stored, not displayed — the record exists; no passage or score is shown (internal_only / metadata_only).
  • Displayed as metadata — citation, provenance, and review status shown; passage/score not shown.
  • Displayed as snippet / full passage — a passage is shown with citation, rights, and edition (evidence_public_snippet_ready / _full_ready).
  • Displayed as metric summary / raw metric — a qualitative summary, or raw scores WITH full run provenance (metric_public_summary_ready / _raw_ready).

Passage & metric display policy

Passages are displayed for scholarly inspection of the relation record. They do not establish reading, contact, source use, dependence, or influence.

Ranking and scoring are retrieval metadata, not evidence of reading, contact, source use, dependence, or influence.

In the current pilot, 46 evidence records are public-display-ready (PI-authorized public-domain passages) and 32 are metadata-only; all 39 metric records are withheld from public display (see the metric-readiness audit). See the relation pages and release history.
